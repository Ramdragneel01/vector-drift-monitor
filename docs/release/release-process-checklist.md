# Release Process Checklist

Repository: vector-drift-monitor
Last updated: 2026-05-05

## Pre-Release
- Confirm CI is green on main.
- Validate changelog entries for user-visible changes.
- Confirm security and dependency scans are acceptable.

## Release
- Tag release with semantic versioning policy.
- Publish release notes with impact and rollback guidance.

## Post-Release
- Monitor health indicators and error budget.
- Capture follow-up issues and hotfix candidates.
